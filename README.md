# API S.Burgers
В коде реализованы тесты ручек API для площадки по заказу бургеров S.Burgers, реализованной в виде сайта-конструктора Бургеров.
В папке test находятся тесты:


- tests_change_user_data.py
Проверяет ручку 'Изменение данных о пользователе'
включает в себя:
Параметризованный тест, проверяющий, что при попытке изменения данных неавторизованным пользоватем, учетные данные не изменяются;
Параметризованный тест, проверяющий вывод сообщения об ошибке при попытке изменения данных неавторизованного пользователя.


- tests_create_order.py
Проверяет ручку 'Создание заказа'
включает в себя негативные проверки:
Создание заказа без авторизации;
Создание заказа без ингредиентов;
Создание заказа с неверным хешем ингредиентов.


- tests_get_user_order.py
Проверяет ручку 'Получение заказов конкретного пользователя'
включает в себя:
Получение заказов авторизованного пользователя;
Попытку получения заказов неавторизованного пользователя.


- tests_login.py
Проверяет ручку 'Авторизация пользователя'
включает в себя:
Проверка логина c передачей валидных данных;
Проверка на отсутствие возможности авторизации с одним неправильным обязательным параметром;
Проверка на отсутствие возможности авторизации без одного обязательного параметра.


- tests_registration_user.py
Проверяет ручку 'Создание пользователя'
включает в себя:
Проверку возможности создания уникального пользователя;
Проверку на отсутствие возможности регистрации двух одинаковых пользователей;
Проверку вывода ожидаемого сообщения 'Email, password and name are required fields' при попытке регистрации пользователя без заполнения обязательных полей.
